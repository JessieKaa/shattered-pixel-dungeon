# PLAN: item-labels-zh

## Goal

在存档编辑器背包(NestedList/NestedObject)中显示 item 中文名称,而非上游 Java 类名。

例如:`com.shatteredpixel.shatteredpixeldungeon.items.armor.ClothArmor` 当前显示为 `ClothArmor`,改后显示为 `布甲(ClothArmor)`。中文名查不到时 fallback 到现有行为(简单类名),UX 不退化。

## Context

### 为什么做

- field-labels-zh 已为字段名(`quantity` / `curCharges` 等)加中文,但 item 标题仍然是裸 Java 类名
- 用户(尤其非开发者)识别"魔弹法杖"远比"WandOfMagicMissile"容易
- 上游 SPD 自带完整 `items_zh.properties`(383 条 `.name=`),资源已就绪,无需自造词典

### 与现有功能的关系

- 复用 `formatClassName`(`composables/useFieldType.ts:29`)的 fallback 行为
- 不动 `__className` 字段本身(只读),仅在渲染层做映射
- 不影响 Bundle 序列化、RawJsonEditor、NumericFields、SlotMetaPanel
- 不引入新依赖

### 关键决策(已敲定)

- **数据来源**:build-time 解析 `core/src/main/assets/messages/items/items_zh.properties`,生成 `frontend/src/data/item-labels-zh.json` 提交进仓
  - 不在 runtime 直接读 `.properties`(避免引入 properties parser)
  - 不发 HTTP 请求动态加载
- **覆盖范围**:只解析 `*.name=` 行(383 条),忽略 `.desc=` / `.ac_*=` / `$xxx.name=` 之外的杂项
  - `$` 嵌套类要保留(如 `bomb$doublebomb.name=一对炸弹`),因为 bundle 的 `__className` 可能是 `Bomb$DoubleBomb`
- **查表失败 fallback**:显示简单类名(原 `formatClassName` 行为),不报错、不告警
- **i18n 切换**:本特性只做中文(与编辑器现状一致,无 i18n 框架),后续若加多语种再重构

### Step 2:写 useItemLabel composable

`tools/save-editor/frontend/src/composables/useItemLabel.ts`:

```ts
import labels from '@/data/item-labels-zh.json'
import { formatClassName } from '@/composables/useFieldType'

// FQCN → properties key
// "com.shatteredpixel.shatteredpixeldungeon.items.wands.WandOfMagicMissile"
//   → "items.wands.wandofmagicmissile"
// "com.shatteredpixel.shatteredpixeldungeon.items.Ankh"
//   → "items.ankh"
// "com.shatteredpixel.shatteredpixeldungeon.items.bombs.Bomb$DoubleBomb"
//   → "items.bombs.bomb$doublebomb"
function toPropKey(className: string): string | null {
  const PREFIX = 'com.shatteredpixel.shatteredpixeldungeon.'
  if (!className.startsWith(PREFIX)) return null
  const tail = className.slice(PREFIX.length)
  // tail 例如 "items.wands.WandOfMagicMissile"
  // 全部小写化(包括 $ 嵌套段)
  return tail.toLowerCase()
}

export interface ItemLabelInfo {
  zh: string | null      // 中文名(查表 miss 时为 null)
  fallback: string       // 简单类名(永远有值)
}

export function itemLabel(cn: unknown): ItemLabelInfo {
  const fallback = formatClassName(cn)
  if (typeof cn !== 'string' || !cn) return { zh: null, fallback }
  const key = toPropKey(cn)
  if (!key) return { zh: null, fallback }
  const zh = (labels as Record<string, string>)[key]
  return { zh: zh ?? null, fallback }
}
```

### Step 5:接入 NestedObject.vue summary

`NestedObject.vue:117-123` 当前:

```ts
function summary(v: unknown): string {
  const t = inferType(v)
  if (t === 'dict') {
    const cn = (v as any)?.__className
    return cn ? formatClassName(cn) : 'dict'
  }
  if (t === 'list') {
    const len = Array.isArray(v) ? v.length : 0
    const classes = Array.isArray(v)
      ? v.map((it) => formatClassName((it as any)?.__className)).join(', ')
      : ''
    return `list[${len}${classes ? ': ' + classes : ''}]`
  }
  return t
}
```

改成:

```ts
import { itemLabel } from '@/composables/useItemLabel'

function summary(v: unknown): string {
  const t = inferType(v)
  if (t === 'dict') {
    const cn = (v as any)?.__className
    if (!cn) return 'dict'
    const info = itemLabel(cn)
    return info.zh ? `${info.zh}(${info.fallback})` : info.fallback
  }
  if (t === 'list') {
    const len = Array.isArray(v) ? v.length : 0
    const classes = Array.isArray(v)
      ? v.map((it) => {
          const info = itemLabel((it as any)?.__className)
          return info.zh ? info.zh : info.fallback
        }).join(', ')
      : ''
    return `list[${len}${classes ? ': ' + classes : ''}]`
  }
  return t
}
```

注意:list summary 只显示中文名(不加括号 fallback),避免太长。

### Finding A:括号风格(PLAN 原文有误)

### Finding B:`%s` 占位符条目

383 条 `.name=` 里,**42 条带 `%s` 占位符**(armor.curses / armor.glyphs / weapon.curses / weapon.enchantments)。这些是 curse / glyph / enchantment 类的"形容词模板",上游在 runtime 把 `%s` 替换成宿主 item 名(如 `反熵布甲`)。Bundle 里 `__className` 直接是 `Glyph$Antimagic` 罕见但**仍可能发生**。直接照搬值会显示 "反熵%s",UX 退化。

建议:extract 脚本里把值末尾的 `%s` 去掉(`.replace(/\s*%s$/, '')`),JSON 里就存 "反熵" / "烈焰"。
